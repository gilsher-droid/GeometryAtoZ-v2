# Fundamatics Learning Engine
## Activity Library

Every lesson is built from reusable activity types.

Each activity should define:

- purpose
- what the student sees
- what the student does
- what the system stores
- completion rule

## Activity Types

### Intro

Purpose: Introduce a lesson or a new idea.

Student action: Read or watch.

Stored data: None.

### Explanation

Purpose: Present a mathematical concept.

Student action: Read, observe, or interact.

Stored data: None.

### Question

Purpose: Encourage thinking before or during learning.

Student action: Write an answer.

Stored data: Student answer.

Completion rule: The current answer is saved.

### Claim

Purpose: Formulate a mathematical statement.

Student action: Write a claim.

Stored data: Student claim.

Completion rule: The current claim is saved.

### Justification

Purpose: Explain why a claim is true.

Student action: Write a justification.

Stored data: Student justification.

Completion rule: The current justification is saved.

### Construction

Purpose: Build a mathematical object.

Student action: Create or manipulate a construction.

Stored data: Construction data.

Completion rule: The required construction is completed.

### Investigation

Purpose: Explore a pattern or relationship.

Student action: Test, compare, change, and observe.

Stored data: Observations or responses.

### Reflection

Purpose: Think about the learning process.

Student action: Write a reflection.

Stored data: Student reflection.

Completion rule: The current reflection is saved.

### Summary

Purpose: Review the main ideas of the lesson.

Student action: Read, respond, or summarize.

Stored data: Optional.
